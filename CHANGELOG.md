# Changelog

## [Unreleased]

### Removed

- Removed the standalone `motion-control/` subproject, removed the desktop-only `obsidian-plugins/knowledgeos-papers` satellite plugin, and dropped the inactive `medical` / `legal` domain skeleton registrations so the repository stays focused on the maintained runtime surfaces.
- Removed the external notebook bridge surface: `khub notebook`, the matching MCP provider/topic-export tools, the `open_notebook` provider adapters, and the sync-manifest SQLite store are gone, while the local notebook workbench helpers remain labs-only.

### Changed

- Extended the core quality-gates tranche with the first follow-up hardening slice. The answer-quality fixture now covers over-citation, caution rewrite blocking, missing factual-claim citations, and mixed non-evidence signals. `index freshness` diagnostics now include source-id coverage when vector metadata can identify canonical paper/vault/web source ids, while still documenting that exact span coverage is not yet available. Production paper compare payloads now attach an additive `comparePacketContract` for ask-v2 `paper_compare` claim-card comparisons, leaving legacy compare payloads untouched.
- Added the core quality-gates tranche above retrieval stabilization. The required `evidence-first-gates` CI job now also runs the deterministic answer-quality fixture gate, compare-packet contract tests, doctor index-freshness coverage, and the card/synthesis/hypothesis inventory guard. `khub doctor --json` now includes an `index freshness` check that compares canonical SQLite source counts with vector metadata coverage without opening the full retrieval stack. The live retrieval-span operator script can now run a baseline-vs-reranker A/B summary, but that remains manual/local because it depends on the active Chroma/vector runtime and local corpus state.
- Added the retrieval quality stabilization gate. A deterministic retrieval-span golden fixture now checks that paper/web/vault queries recover expected source ids and text terms without live providers, non-evidence retrieval rows remain signal-only, and `docs/store_authority_inventory.json` is enforced by SQL introspection so lifecycle-required derivative tables keep `source_content_hash`, `stale`, `stale_reason`, and `invalidated_at` columns. The required `evidence-first-gates` CI job now runs those protections too.
- Added the next evidence-first hardening tranche across epistemic review history, ontology contributor invalidation, answer verification, and CI. `belief_review` / `decision_review` now create successor rows linked by `supersedes` / `superseded_by` instead of overwriting review state in place, default belief/decision lists hide superseded history, concept entities now accumulate `contributor_hashes` from derived ontology claim/relation writes and only go stale when all live contributors for an affected hash set are gone, answer verification now fails `contradicts_rejected_belief` or `signal_only_grounding` cases instead of treating them as ordinary grounded success, and a new required CI job runs a narrow evidence-first golden/contract gate (`tests/test_evidence_first_golden.py`, answer-contract runtime, verification guards, epistemic supersede, mixed-store lifecycle).
- Added the first evidence-vs-signal guard in the answer contract. `build_evidence_packet_contract(...)` now excludes non-evidence mixed-store spans from contract `spans`, `build_answer_contract(...)` refuses to emit citations for blocked mixed-store sources (`belief`, `decision`, `outcome`, `ontology`, `learning_*`, `memory_relation`, `entity_merge`, `entity_split`), and the dropped rows are recorded under `retrievalSignals` instead of being treated as citation-grade grounding.
- Added the first mixed-store lifecycle/filter tranche on top of the earlier schema prep. `ontology_claims`, `ontology_relations`, legacy `kg_relations`, `learning_graph_{nodes,edges,paths,resource_links}`, and `memory_relations` now carry lifecycle columns where needed, default list/read paths exclude stale derivative rows, and `origin='manual'` memory/learning edges remain visible instead of being auto-hidden. Source-linked mixed derivatives are invalidated only on explicit source-document hash changes (`note` / `paper` / `web` / `vault`), not on `document_memory` refresh, so paper/document-memory rebuilds no longer silently wipe claim-first or relation-first retrieval candidates.
- Added table-level schema prep for the next mixed-store lifecycle tranche without changing runtime behavior. `memory_relations`, ontology claims/relations/legacy KG relations, and learning-graph edges/resource links now reserve an `origin` marker (`derived|manual|pending`); ontology concept/entity projections now reserve `contributor_hashes`; epistemic beliefs and decisions now reserve `supersedes` / `superseded_by`. The new ADR `docs/adr/2026-04-24-mixed-store-schema-prep.md` records the table-level authority split that will govern later lifecycle and retrieval filtering work.
- Split the verification fallback semantics more precisely. When answer verification resolves to the local heuristic path (`route=local`), the runtime still produces heuristic `caution` results; when verification is unavailable because the route is truly `fallback-only` from missing config, the runtime now returns `status=skipped` and preserves the original answer instead of forcing a conservative fallback.
- Added a shared Python/Foundry policy conformance fixture and enforced it on both sides. Python `evaluate_policy_gate` and Foundry `LocalPolicyEngine` now agree on declared P0, nested `policy.classification=P0`, detected P0, structured P1 JSON content, summary-shaped P2, and public-url P3 cases; Foundry also treats payload-declared classification as advisory input instead of relying only on the caller-supplied classification argument.
- Closed the remaining Opus P0 answer-contract gaps on the split branch. `VerificationVerdict.rewriteAllowed` now opens only on `pass`, caution/missing-conflict-language cases route to conservative fallback instead of the rewrite LLM, `EvidencePacket` strict mode excludes low-provenance or stale spans and treats missing policy classification as `UNKNOWN` with external calls fail-closed, and `AnswerContract.coverageRatio` is now based on claim-to-citation mapping rather than raw citation counts.
- Cleaned the Opus hardening split stack for release hygiene. The split branch now removes tracked `eval/knowledgeos/review/` runtime review artifacts, ignores that generated review directory going forward, fixes the narrow CI contract suite so it no longer references a missing `tests/test_status_cmd.py`, converts the continuous-sync launchd file into a placeholder template, and makes `scripts/run_continuous_sync.sh` local-path-free with `.env`, `--apply`, and `--allow-external` all explicitly opt-in.
- Added runtime enforcement for the evidence-first answer contract. `khub ask` payloads now carry additive `EvidencePacket`, `AnswerContract`, and `VerificationVerdict` contract objects with source-hash/span/citation fields that validate against registered schemas; answer verification now treats unsupported or uncertain-only claims as a gate to conservative fallback instead of an LLM rewrite opportunity; ask-v2 weak/missing evidence with unsupported fields now abstains before answer generation; `khub agent run` reclassifies returned artifacts through the Python P0 policy gate instead of trusting Foundry-provided metadata; and Foundry's local policy engine now treats declared artifact classification as advisory by re-scanning payloads and upgrading to the most sensitive detected class before verify/writeback.
- Closed the remaining authority and RAG compatibility gaps in that answer-contract tranche. The Python MCP policy gate now returns the most sensitive non-P0 classification between declared metadata and detected payload shape, Foundry direct runtime envelopes redact P0 verify-blocked artifact bodies, the answer orchestrator again preserves direct helper/cache behavior while honoring test-visible router overrides, topology expansion again honors the searcher topology hook, and concept-explainer answers no longer single-scope to a query-plan anchor when the evidence policy says `singleScopeRequired=false`.
- Replayed the Opus answer-contract hardening stack into a dedicated clean sibling branch, `opus-architecture-hardening-clean-20260423`, by applying the exact checkpoint buckets `01`-`07` plus the `00`, `09`, `10`, `11`, and `12` overlays. The replay fixed the broad `.gitignore` `src/` and `data/` patterns so nested `foundry-core/src` and packaged ontology profiles are tracked, carried the missing paper-memory eval fixture and default ontology profile YAMLs, and fixed paper answer diagnostics so deprecated requested mode tokens such as `prefilter` remain visible while effective routing still normalizes to `compat`.
- Hardened the default RAG answer evidence and rewrite gate. Answer evidence items now carry additive source-trace diagnostics (`source_ref`, `source_id`, `source_content_hash`, `span_locator`, `snippet_hash`, `evidence_kind`, and derivative-source metadata when present), so raw spans, raw sources, derived anchors, and memory hints are distinguishable in payloads. Verification failures with unsupported claims no longer call the LLM rewrite route first; they mark `requiresConservativeFallback` and flow into the conservative fallback path, while conflict-language-only caution can still use rewrite.
- Added the first derivative lifecycle contract for semantic stores. `document_memory_units` now records `source_content_hash` plus stale diagnostics, paper memory/cards, paper section cards, web cards, vault cards, and claim cards now carry the same lifecycle columns, and document-memory replacement marks dependent derivatives stale when their recorded source hash differs. Card/claim search paths exclude stale derivatives by default while direct getters still expose stale rows for inspection/rebuild.
- Extended source invalidation into raw source writes and vector indexing. Note/web/vault writes now preserve or compute `source_content_hash` and mark dependent derivative cards stale when hashes differ or the derivative has no recorded hash; paper/vault/web indexing paths write source-hash plus `stale=0` vector metadata and replace prior document chunks before reindexing; vector semantic and lexical search now exclude rows with explicit stale metadata by default.

Raw source hash changes now mark stale `document_memory_units` as well as downstream cards/claims, paper document-memory rows record the hash of the selected source file/content, and hidden `khub paper source-freshness` audits current paper source hashes against stored document-memory hashes. Vault query planning now marks DB-confirmed unresolved `vault:` ids or Markdown paths as `answer_mode=abstain` with `vault_scope_missing`, and ask-v2 vault card selection stops instead of broadening to unrelated vault evidence when a hard-scoped note/path cannot be resolved.
